# 规则引擎深度解析

规则是梅菲斯特的核心驱动力。本章节详细解释规则的条件、动作、互斥组和骰子。

## 一、规则格式

```text
[规则名] if 条件 -> 动作
```

- **规则名**：唯一标识符（用于调试和互斥组）。
- **条件**：决定规则是否触发的逻辑表达式。
- **动作**：触发后执行的操作。

## 二、条件表达式

### 1. 原子条件

- 变量比较：`情绪 == "暴怒"`、`生命值 != 0`
- 数字比较：`堕落指数 > 70`、`生命值 >= 20`
- 包含判断：`输入包含 "光之国"`（检查变量是否包含子串）

### 2. 逻辑运算

- `&&`（且）：`情绪 == "暴怒" && 堕落指数 > 70`
- `||`（或）：`roll(1d100) > 80 || 情绪 == "疯狂"`
- 括号分组：`(情绪 == "暴怒" || 情绪 == "疯狂") && 堕落指数 > 70`

### 3. 骰子表达式

- 格式：`roll(数量d面数)`
- 示例：`roll(1d100) > 80`（百分骰大于 80）
- 注意：骰子表达式**只在条件中使用**，动作中的骰子仅输出结果。

## 三、动作类型

### 1. 注入动作

```meph
[知识] if 输入包含 "光之国" -> 注入 "{角色名}的故乡是光之国"
```

- 将内容注入上下文（不直接显示给用户，但 LLM 会将其融入叙事）。
- 支持 `{变量}` 替换。

### 2. LLM 生成动作（高级）

```meph
[创意] if 输入包含 "想象" -> LLM: "用诗意的语言描述当前场景"
```

- 直接给 LLM 发送特定的生成指令。
- 忽略普通规则消息，仅输出 LLM 生成的内容。

### 3. 普通动作

```meph
[嘲讽] if 输入包含 "光之国" -> 先嘲讽再回应
```

- 直接在终端显示动作描述。

## 四、互斥组（Mutual Exclusion）

在规则动作前添加 `[group:组名]`，同一组中只有第一个匹配的规则会执行。

**示例：**

```meph
[攻击] if 情绪 == "暴怒" && 堕落指数 > 70 -> [group:combat] 全力攻击
[攻击] if 情绪 == "暴怒" -> [group:combat] 普通攻击
[防御] if 情绪 == "恐惧" -> [group:combat] 防御姿态
```

如果 `堕落指数 > 70`，只会执行“全力攻击”，忽略后续同组规则。

## 五、执行顺序与优先级

1. **顺序执行**：规则按在 `【规则】` 区块中的书写顺序从上到下检查。
2. **全部执行**：非互斥组的规则会全部执行（如注入规则）。
3. **互斥组**：同组只执行第一个匹配项，后面的被跳过。

## 六、注意事项

- **状态变量**（如 `情绪`）目前在规则触发后不会自动更新，需要在叙事中手动调整（或通过扩展示来支持）。
- **输入变量**：`输入` 是特殊的内部变量，代表用户最后一次输入的内容。
