# 梅菲斯特规则引擎深度解析

梅菲斯特的规则引擎是驱动叙事的核心。它读取 `.meph` 契约中的 `【规则】` 区块，在运行时匹配条件、执行动作，并结合 LLM 生成自然叙事。

---

## 一、规则语法

### 基本格式

```
[规则名] if 条件 -> 动作
```

### 规则名

- 用 `[]` 包裹，不能为空。
- 用于标识规则，便于调试和日志显示。

### 条件（Condition）

条件决定了规则何时触发。支持以下语法：

| 语法               | 说明                     | 示例                                |
| ------------------ | ------------------------ | ----------------------------------- |
| `包含 "关键词"`    | 用户输入包含指定关键词   | `包含 "攻击"`                       |
| `不包含 "关键词"`  | 用户输入不包含指定关键词 | `不包含 "光之国"`                   |
| `状态.键 > 值`     | 状态值大于指定值         | `状态.堕落指数 > 80`                |
| `状态.键 >= 值`    | 状态值大于等于指定值     | `状态.生命值 >= 0`                  |
| `状态.键 < 值`     | 状态值小于指定值         | `状态.理智值 < 30`                  |
| `状态.键 <= 值`    | 状态值小于等于指定值     | `状态.体力 <= 10`                   |
| `状态.键 == 值`    | 状态值等于指定值         | `状态.情绪 == "暴怒"`               |
| `状态.键 != 值`    | 状态值不等于指定值       | `状态.存活 != false`                |
| `条件1 && 条件2`   | 与运算（同时满足）       | `包含 "攻击" && 状态.堕落指数 > 80` |
| `条件1 \|\| 条件2` | 或运算（任意满足）       | `包含 "攻击" \|\| 包含 "战斗"`      |

**注意**：`状态.键` 中的键名必须与 `【状态】` 区块中定义的键完全一致。

### 动作（Action）

动作决定了规则触发后执行什么操作。

| 动作类型     | 格式                | 说明                                      |
| ------------ | ------------------- | ----------------------------------------- |
| **注入**     | `注入 "消息"`       | 将消息追加到记忆，由 LLM 自然融入后续叙事 |
| **LLM 生成** | `LLM: 指令`         | 向 LLM 发送生成指令                       |
| **状态修改** | `状态.键 = 值`      | 更新运行时状态值                          |
| **普通动作** | `任意文本`          | 作为指令传给 LLM 执行                     |
| **互斥组**   | `[group:组名] 动作` | 同一组只触发第一个匹配的规则              |

### 互斥组

互斥组确保同一组内最多只有一条规则被触发。

```meph
[攻击] if 包含 "攻击" -> [group:combat] 注入 "贝利亚发动了猛烈的攻击"
[防御] if 包含 "防御" -> [group:combat] 注入 "贝利亚摆出了防御姿态"
[逃跑] if 包含 "逃跑" -> [group:combat] 注入 "贝利亚选择了撤退"
```

当用户输入包含 `"攻击"` 时，只有 `[攻击]` 被触发；输入包含 `"防御"` 时，只有 `[防御]` 被触发。

## 二、规则执行流程

```
用户输入 → 按顺序匹配规则
         ├── 匹配成功 → 执行动作
         │   ├── 注入 → 追加到记忆，继续调用 LLM 叙事
         │   ├── LLM → 调用 LLM 生成响应
         │   ├── 状态修改 → 更新状态，返回确认
         │   └── 普通动作 → 作为指令调用 LLM
         └── 无匹配 → 调用 LLM 自由叙事（若 LLM 可用）
         └── 无匹配且无 LLM → 返回默认响应
```

## 三、完整示例

```meph
【角色名】
贝利亚奥特曼

【状态】
- 堕落指数: 85
- 情绪: 暴怒

【规则】
# 互斥组 - 战斗动作
[攻击] if 包含 "攻击" || 包含 "战斗" -> [group:combat] 注入 "贝利亚发动了猛烈的攻击"
[防御] if 包含 "防御" || 包含 "防守" -> [group:combat] 注入 "贝利亚摆出了防御姿态"
[逃跑] if 包含 "逃跑" || 包含 "撤退" -> [group:combat] 注入 "贝利亚选择了撤退"

# 注入背景信息
[光之国背景] if 包含 "光之国" -> 注入 "{角色名}的故乡是光之国，也是他最大的仇恨来源"
[雷布朗多] if 包含 "雷布朗多" || 包含 "黑暗" -> 注入 "雷布朗多在你体内低语：力量才是唯一真理，{角色名}"

# LLM 生成
[嘲讽] if 包含 "光之国" -> LLM: 以讽刺的语气回应
```

## 六、最佳实践

1. **规则顺序即优先级**：将最可能的规则放在前面，减少不必要的计算。
2. **使用互斥组管理互斥规则**：同一组内规则按优先级排序。
3. **注入内容要自然**：注入的内容会被 LLM 融入叙事，应使用叙事化语言。
4. **条件要具体**：避免过于宽泛的条件导致意外触发。
5. **结合状态驱动**：使用 `状态.键` 条件让角色状态影响叙事发展。
