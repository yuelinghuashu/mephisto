# 梅菲斯特语法手册

`.meph` 文件是梅菲斯特引擎的契约文件。它由多个 **区块（Block）** 组成，每个区块通过 `【区块名】` 定义。

---

## 一、8 个用户区块（由创作者书写）

| 区块           | 类型       | 重要性     | 说明                       |
| -------------- | ---------- | ---------- | -------------------------- |
| `【角色名】`   | 单行文本   | **★ 必选** | 角色的唯一标识             |
| `【锚点】`     | 键值对列表 | **★ 推荐** | 核心人格，永不压缩         |
| `【世界观】`   | 多行文本   | 可选       | 故事发生的世界背景         |
| `【角色背景】` | 多行文本   | 可选       | 角色的历史与起源           |
| `【开局场景】` | 多行文本   | 可选       | 对话开始时的初始场景描述   |
| `【状态】`     | 键值对列表 | 可选       | 动态变量（情绪、生命值等） |
| `【规则】`     | 规则列表   | 可选       | 条件-动作规则，驱动交互    |
| `【校验】`     | 键值对列表 | 可选       | 后置检查配置（保留中）     |

## 二、2 个系统区块（由程序自动生成）

| 区块       | 类型       | 说明                           |
| ---------- | ---------- | ------------------------------ |
| `【记忆】` | 纯文本列表 | 程序自动从对话中提取的长期记忆 |
| `【历史】` | 键值对列表 | 程序自动记录的对话历史         |

> **注意**：`【记忆】` 和 `【历史】` 区块由系统在运行时自动生成和维护，**创作者无需手动书写**。它们会出现在子版文件（`story_child.meph`）中，用于持久化故事进度。

---

## 三、区块写法详解

### 1. 文本区块（支持变量替换）

适用于：`【角色名】`、`【世界观】`、`【角色背景】`、`【开局场景】`

- 直接书写文本，保留空行。
- 支持 `{变量名}` 插值。
- 变量来源：`【状态】` 中的键 + `【角色名】` 的值。

**示例：**

```meph
【角色名】
暗影行者

【世界观】
{角色名}是暗影森林的守护者。

【开局场景】
你站在{位置}中，{角色名}的记忆开始苏醒。
```

### 2. 键值对列表区块

适用于：`【锚点】`、`【状态】`、`【校验】`

- 格式：`- 键: 值`
- 支持中文冒号（`：`）和英文冒号（`:`）。
- 键名自动成为全局变量（在 `【状态】` 下）。

**示例：**

```meph
【状态】
- 情绪: 暴怒
- 生命值：100       # 中文冒号同样有效
- 堕落指数: 85
```

### 3. 规则列表区块

适用于：`【规则】`

- 格式：`[规则名] if 条件 -> 动作`
- 规则按书写顺序从上到下执行（顺序即优先级）。

**支持的动作类型：**

| 动作类型     | 示例          | 说明                                 |
| ------------ | ------------- | ------------------------------------ |
| **注入**     | `注入 "内容"` | 将内容作为叙事背景注入（不直接显示） |
| **LLM 生成** | `LLM: 提示词` | 向 LLM 发送特定的生成指令            |
| **骰子**     | `roll(1d100)` | 掷骰并输出结果                       |
| **普通动作** | `嘲讽对手`    | 直接显示动作描述                     |

**示例：**

```meph
【规则】
[攻击] if 输入包含 "攻击" -> [group:combat] 注入 "贝利亚发动了猛烈的攻击"
[防御] if 输入包含 "防御" -> [group:combat] 注入 "贝利亚摆出了防御姿态"
[嘲讽] if 输入包含 "光之国" -> 先嘲讽再回应
[光之国背景] if 输入包含 "光之国" -> 注入 "{角色名}的故乡是光之国"
```

## 四、注释与空白

- **注释**：以 `#` 开头的行会被完全忽略。
- **空行**：在文本区块中会被保留（保持段落格式），在列表/规则区块中会被忽略。

**示例：**

```meph
【角色名】
贝利亚奥特曼   # 这是单行文本

# 下面这行会被忽略
【锚点】
- 核心信念: "力量就是一切"  # 行内注释仅支持全行 #
```

---

## 五、全局变量系统

`【状态】` 中的所有键值对和 `【角色名】` 的值，自动成为全局变量。

- 在文本区块中用 `{变量名}` 引用。
- 在规则动作中自动替换（如注入内容）。

**示例：**

```meph
【角色名】
贝利亚

【状态】
- 堕落指数: 85

【规则】
[警告] if 堕落指数 > 80 -> 注入 "警告：{角色名}的堕落指数已达 {堕落指数}"
```

## 六、子版存档与分支

梅菲斯特会自动保存运行时数据到子版文件中。

### 文件命名规则

| 场景     | 命令                                     | 生成文件               |
| -------- | ---------------------------------------- | ---------------------- |
| 默认子版 | `go run main.go story.meph`              | `story_child.meph`     |
| 指定分支 | `go run main.go -branch dark story.meph` | `story_dark.meph`      |
| 加载分支 | `go run main.go story_dark.meph`         | 更新 `story_dark.meph` |

### 子版文件结构

子版文件包含：

- **所有母版静态区块**（角色名、世界观、锚点、规则等）
- **最新的 `【状态】`**（当前情绪、生命值、位置等）
- **系统生成的 `【记忆】`**（程序自动从对话中提取的长期记忆）
- **系统生成的 `【历史】`**（程序自动记录的对话历史，保留最近 N 轮）

子版文件独立可运行，可直接加载继续故事。

---

## 七、交互命令

在对话界面中，支持以下命令：

| 命令                  | 说明               |
| --------------------- | ------------------ |
| `quit` / `exit` / `q` | 退出对话并自动保存 |

按 `Ctrl+C` 也可中断当前 LLM 生成或退出程序。
