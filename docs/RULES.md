# 梅菲斯特规则引擎深度解析

梅菲斯特的规则引擎是驱动叙事的核心。它读取 `.meph` 契约中的 `【规则】` 区块，在运行时匹配条件、执行动作，并结合 LLM 生成自然叙事。

---

## 一、规则语法

### 基本格式

```markdown
[规则名] if 条件 -> 动作
```

### 规则名

- 用 `[]` 包裹，不能为空。
- 用于标识规则，便于调试和日志显示。

### 条件（Condition）

条件决定了规则何时触发。支持以下语法：

| 语法                | 说明                     | 示例                                  |
| ------------------- | ------------------------ | ------------------------------------- |
| `包含 "关键词"`     | 用户输入包含指定关键词   | `包含 "攻击"`                         |
| `不包含 "关键词"`   | 用户输入不包含指定关键词 | `不包含 "满足"`                       |
| `状态.键 > 值`      | 状态值大于指定值         | `状态.灵魂完整度 > 80`                |
| `状态.键 >= 值`     | 状态值大于等于指定值     | `状态.生命值 >= 0`                    |
| `状态.键 < 值`      | 状态值小于指定值         | `状态.理智值 < 30`                    |
| `状态.键 <= 值`     | 状态值小于等于指定值     | `状态.体力 <= 10`                     |
| `状态.键 == 值`     | 状态值等于指定值         | `状态.情绪 == "永不满足"`             |
| `状态.键 != 值`     | 状态值不等于指定值       | `状态.存活 != false`                  |
| `条件1 && 条件2`    | 与运算（同时满足）       | `包含 "攻击" && 状态.灵魂完整度 > 50` |
| `条件1 \|\| 条件2`  | 或运算（任意满足）       | `包含 "攻击" \|\| 包含 "战斗"`        |
| `roll(1d100)`       | 骰子表达式（默认 50%）   | `roll(1d100)`                         |
| `roll(1d100) >= 80` | 骰子表达式（自定义阈值） | `roll(1d100) >= 80`                   |

**注意**：

- `状态.键` 中的键名必须与 `【状态】` 区块中定义的键完全一致。
- 骰子默认阈值 = 骰子数 × 面数 / 2（50% 成功率）。
- 支持自定义阈值操作符：`>=` `>` `<=` `<` `==` `!=`。

### 动作（Action）

动作决定了规则触发后执行什么操作。

| 动作类型             | 格式            | 说明                                      |
| -------------------- | --------------- | ----------------------------------------- |
| **注入**             | `注入 "消息"`   | 将消息追加到记忆，由 LLM 自然融入后续叙事 |
| **状态修改（简单）** | `状态.键 = 值`  | 更新运行时状态值                          |
| **状态修改（复合）** | `状态.键 += 值` | 加法赋值（仅数字类型）                    |
|                      | `状态.键 -= 值` | 减法赋值                                  |
|                      | `状态.键 *= 值` | 乘法赋值                                  |
|                      | `状态.键 /= 值` | 除法赋值                                  |
| **普通动作**         | `任意文本`      | 有 LLM 时作为指令调用，否则直接返回该文本 |

复合赋值运算结果自动保持原类型：`int` → `int`，`float64` → `float64`。
`int` 除法结果直接舍去小数。

### 互斥组（规则修饰符）

互斥组确保同一组内最多只有一条规则被触发。

```meph
[深入追问] if 包含 "为什么" || 包含 "为何" -> [group:dialogue] 用学术口吻深入分析，引用经典
[转移话题] if 包含 "够了" || 包含 "停下" -> [group:dialogue] 沉默片刻，转而评论窗外的夜色
```

当用户输入包含 `"为什么"` 时，只有 `[深入追问]` 被触发；输入包含 `"够了"` 时，只有 `[转移话题]` 被触发。

## 二、规则执行流程

### 两阶段执行

梅菲斯特采用**两阶段规则执行**，自动区分"被动规则"和"主动规则"：

```
用户输入
  │
  ├── 阶段一：被动规则批量执行
  │   ├── 匹配所有「注入」和「状态修改」规则
  │   ├── 所有匹配的规则都依次执行（互斥组内只执行第一个）
  │   └── 每条规则单独读取最新状态，支持链式状态变化
  │
  ├── 阶段二：主动规则互斥匹配
  │   ├── 匹配「普通文本」等产生直接输出的规则
  │   ├── 只执行第一个匹配的规则（含互斥组）
  │   └── 使用阶段一更新后的状态进行判定
  │
  └── 结果
      ├── 有主动规则匹配 → 执行该规则动作
      ├── 仅被动规则匹配 → 调用 LLM 继续叙事
      └── 无任何规则匹配 → 调用 LLM 自由叙事（无 LLM 则返回默认响应）
```

**被动规则**（状态修改 `状态.` / 注入记忆 `注入 `）可批量执行，因为它们只产生副作用而不直接输出。多条规则同时匹配时全部生效，不会被"截胡"。

**主动规则**（普通文本/LLM 指令）互斥执行，同一轮只触发第一个匹配的规则。

### 被动规则链式变化示例

```meph
# 以下两条规则若条件同时满足，都会被执行：
[堕落加深] if 包含 "力量" && 状态.堕落指数 > 50 -> 状态.堕落指数 += 10
[暴走检测] if 状态.堕落指数 > 90 -> 状态.情绪 = "暴走"

# 输入 "我感觉到了更强的力量！"
# → 阶段一：堕落加深 执行（堕落指数 85 → 95）
# → 阶段一：暴走检测 也执行（堕落指数 95 > 90 → 情绪变为 "暴走"）
```

### 骰子判定流程

```
用户输入 → 条件包含 roll(...)
         ├── 本地掷骰 → 判定是否达到阈值（决定规则是否触发）
         ├── 重新掷骰 → 生成骰子结果描述（注入 LLM 指令中）
         └── LLM 收到指令 → 根据骰子结果影响故事走向
```

## 三、完整示例

```meph
【角色名】
浮士德

【状态】
- 灵魂完整度：50
- 情绪：永不满足

【规则】
# 互斥组 - 对话驱动
[深入追问] if 包含 "为什么" || 包含 "为何" -> [group:dialogue] 用学术口吻深入分析，引用经典
[转移话题] if 包含 "够了" || 包含 "停下" -> [group:dialogue] 沉默片刻，转而评论窗外的夜色

# 注入背景信息
[梅菲斯特回声] if 包含 "契约" || 包含 "灵魂" -> 注入 "梅菲斯特的声音在{角色名}耳边低语：这就是你想要的，代价你可想好了"

# 骰子判定（自定义阈值）
[灵魂流逝] if 包含 "代价" && roll(1d100) >= 80 -> 注入 "{角色名}低头看着自己的双手，仿佛能看见什么东西正在一丝丝流逝"
```

## 五、最佳实践

1. **规则顺序即优先级**：将最可能的规则放在前面，减少不必要的计算。
2. **使用互斥组管理互斥规则**：同一组内规则按优先级排序。
3. **注入内容要自然**：注入的内容会被 LLM 融入叙事，应使用叙事化语言。
4. **条件要具体**：避免过于宽泛的条件导致意外触发。
5. **结合状态驱动**：使用 `状态.键` 条件让角色状态影响叙事发展。
6. **合理使用骰子**：骰子表达式提供随机判定，自定义阈值控制触发概率。
