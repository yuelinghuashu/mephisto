# 梅菲斯特语法手册

`.meph` 文件是梅菲斯特引擎的契约文件。它由多个 **区块（Block）** 组成，每个区块通过 `【区块名】` 定义。

## 一、9 个标准区块

| 区块           | 类型       | 重要性     | 说明                       |
| -------------- | ---------- | ---------- | -------------------------- |
| `【角色名】`   | 单行文本   | **★ 必选** | 角色的唯一标识             |
| `【锚点】`     | 键值对列表 | **★ 推荐** | 核心人格，永不压缩         |
| `【世界观】`   | 多行文本   | 可选       | 故事发生的世界背景         |
| `【角色背景】` | 多行文本   | 可选       | 角色的历史与起源           |
| `【开局场景】` | 多行文本   | 可选       | 对话开始时的初始场景       |
| `【状态】`     | 键值对列表 | 可选       | 动态变量（情绪、生命值等） |
| `【规则】`     | 规则列表   | 可选       | 条件-动作规则              |
| `【校验】`     | 键值对列表 | 可选       | 后置检查配置               |
| `【记忆】`     | 键值对列表 | 可选       | 记忆系统参数               |

## 二、区块写法详解

### 1. 文本区块（支持变量替换）

适用于：`【角色名】`、`【世界观】`、`【角色背景】`、`【开局场景】`

- 直接书写文本，保留空行。
- 支持 `{变量名}` 插值。
- 变量来源：`【状态】` 中的键 + `【角色名】` 的值。

**示例：**

```meph
【角色名】
暗影行者

【世界观】
{角色名}是暗影森林的守护者。

【开局场景】
你站在{位置}中，{角色名}的记忆开始苏醒。
```

### 2. 键值对列表区块

适用于：`【锚点】`、`【状态】`、`【校验】`、`【记忆】`

- 格式：`- 键: 值`
- 支持中文冒号（`：`）和英文冒号（`:`）。
- 键名自动成为全局变量（在 `【状态】` 下）。

**示例：**

```meph
【状态】
- 情绪: 暴怒
- 生命值：100       # 中文冒号同样有效
- 堕落指数: 85
```

### 3. 规则列表区块

适用于：`【规则】`

- 格式：`[规则名] if 条件 -> 动作`
- 规则按书写顺序从上到下执行（顺序即优先级）。

**示例：**

```meph
【规则】
[攻击] if 输入包含 "攻击" -> 全力攻击
[嘲讽] if 输入包含 "光之国" -> 先嘲讽再回应
```

## 三、注释与空白

- **注释**：以 `#` 开头的行会被完全忽略。
- **空行**：在文本区块中会被保留（保持段落格式），在列表/规则区块中会被忽略。

**示例：**

```meph
【角色名】
贝利亚奥特曼   # 这是单行文本

# 下面这行会被忽略
【锚点】
- 核心信念: "力量就是一切"  # 行内注释仅支持全行 #
```

## 四、全局变量系统

`【状态】` 中的所有键值对和 `【角色名】` 的值，自动成为全局变量。

- 在文本区块中用 `{变量名}` 引用。
- 在规则动作中自动替换（如注入内容）。

**示例：**

```meph
【角色名】
贝利亚

【状态】
- 堕落指数: 85

【规则】
[警告] if 堕落指数 > 80 -> 注入 "警告：{角色名}的堕落指数已达 {堕落指数}"
```
