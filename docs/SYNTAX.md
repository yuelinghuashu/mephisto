# 梅菲斯特语法手册

`.meph` 文件是梅菲斯特引擎的契约文件。它由多个 **区块（Block）** 组成，每个区块通过 `【区块名】` 定义。

## 一、9 个标准区块

| 区块           | 类型       | 重要性     | 说明                       |
| -------------- | ---------- | ---------- | -------------------------- |
| `【角色名】`   | 单行文本   | **★ 必选** | 角色的唯一标识             |
| `【锚点】`     | 键值对列表 | **★ 推荐** | 核心人格，永不压缩         |
| `【世界观】`   | 多行文本   | 可选       | 故事发生的世界背景         |
| `【角色背景】` | 多行文本   | 可选       | 角色的历史与起源           |
| `【开局场景】` | 多行文本   | 可选       | 对话开始时的初始场景       |
| `【状态】`     | 键值对列表 | 可选       | 动态变量（情绪、生命值等） |
| `【规则】`     | 规则列表   | 可选       | 条件-动作规则              |
| `【记忆】`     | 纯文本列表 | 可选       | 长期记忆（系统自动生成）   |
| `【历史】`     | 键值对列表 | 可选       | 对话历史（系统自动生成）   |

> **说明**：`【记忆】` 和 `【历史】` 是**系统区块**，由程序自动生成和维护，通常不需要用户手动书写。但如果你需要手动编辑子版文件，可以按上述格式修改。
>
> **区块标题格式**：必须严格以 `【` 开头、以 `】` 结尾。不完整格式（如 `【锚点` 或 `锚点】`）会被识别为格式错误并报错。

---

## 二、区块写法详解

### 1. 文本区块（支持变量替换）

适用于：`【角色名】`、`【世界观】`、`【角色背景】`、`【开局场景】`

- 直接书写文本，保留空行。
- 支持 `{角色名}` 插值替换。
- 变量来源：`【角色名】` 区块的值。

**示例：**

```meph
【角色名】
浮士德

【世界观】
故事发生在 16 世纪的德意志，这是一个神学与科学交织的时代。
大学中充斥着经院哲学的空洞争论，真正的新知被压制。
世界由上帝、天使、魔鬼与凡人共同构成，地狱与天堂是真实存在的维度。
梅菲斯特是地狱的使者，擅长以言语和契约诱捕人类灵魂。
灵魂交易在此世界是真实的契约——任何灵魂契约一旦签订，绝无撤回的可能。

【开局场景】
浮士德是一位学识渊博的学者，精通哲学、医学、法学和神学。
但他对人类知识的极限感到绝望——穷尽一生所学，仍无法触及世界的本质。
绝望中，他与梅菲斯特签订了契约：用灵魂换取在世上的无限体验。
```

### 2. 键值对列表区块

适用于：`【锚点】`、`【状态】`、`【历史】`

- 格式：`- 键: 值`
- 支持中文冒号（`：`）和英文冒号（`:`）。

**示例：**

```meph
【状态】
- 灵魂完整度：50
- 情绪：永不满足
- 位置：书斋
```

### 3. 纯文本列表区块

适用于：`【记忆】`

- 格式：`- 条目内容`
- 不解析键值对，整行内容作为字符串值。

**示例：**

```meph
【记忆】
- 深夜。书斋中烛火摇曳，桌上的书籍堆积成山。
- 浮士德站在窗边，望着窗外的月光。
- 桌角放着一份契约书，墨迹还未干透。
```

### 4. 规则列表区块

适用于：`【规则】`

- 格式：`[规则名] if 条件 -> 动作`
- 规则按书写顺序从上到下执行（顺序即优先级）。

**示例：**

```meph
【规则】
[新体验] if 包含 "追求" -> 注入 "{角色名}感到心中燃起新的渴望，没有什么能阻止他去亲自体验这一切"
[梅菲斯特] if 包含 "梅菲斯特" || 包含 "契约" -> 注入 "梅菲斯特的声音在{角色名}耳边低语：'这就是你想要的吗？代价你可想好了。'"
```

**复合动作**：用 `&&` 串联多个子动作，依次执行。例如：

```
[双重注入] if 包含 "命运" -> 注入 "烛火猛地窜高" && 注入 "{角色名}感到命运在向他招手"
```

> 支持注入、状态修改、普通文本等任意动作类型的组合。

## 三、注释与空白

- **注释**：以 `#` 开头的行会被完全忽略（前面允许有空格）。
- **区块差异规则**：
  - **文本区块**（世界观、角色背景、开局场景）：允许行内 `#`，可用作话题标签、编号等自由文本。
  - **规则区块**：禁止行内 `#`（`#` 仅能出现在行首作为注释）。
  - **键值对区块**（锚点、状态）：禁止行内 `#`。
  - **记忆/历史区块**：允许行内 `#`（系统自动生成，内容不可控）。
- **空行**：在文本区块中会被保留（保持段落格式），在列表/规则区块中会被忽略。

**示例：**

```meph
【角色名】
浮士德

# 下面这行会被忽略
  # 带空格的注释也允许
【锚点】
- 核心信念：知识高于一切

【世界观】
这个世界充满了 #魔法# 与 #科技# 的结合  ← 文本区块中允许
```

## 四、全局变量系统

`【角色名】` 的值是唯一的全局变量 `{角色名}`，在文本区块和规则动作中自动替换。

**示例：**

```meph
【角色名】
浮士德

【规则】
[背景] if 包含 "梅菲斯特" -> 注入 "{角色名}与梅菲斯特签订了契约，以灵魂换取无限体验"
```

## 五、骰子表达式

骰子表达式可在规则条件中使用，提供随机判定：

| 语法                | 说明                              |
| ------------------- | --------------------------------- |
| `roll(1d100)`       | 默认阈值 50%（结果 >= 50 为成功） |
| `roll(1d100) >= 80` | 自定义阈值（结果 >= 80 为成功）   |
| `roll(2d6)`         | 两个 6 面骰，默认阈值 7           |
| `roll(1d20) <= 5`   | 结果 <= 5 为成功                  |

判定为成功后，骰子数值会传递给 LLM 影响故事走向。

## 六、自定义区块（环境变量扩展）

梅菲斯特支持通过环境变量 `MEPHISTO_EXTRA_BLOCKS` 添加自定义区块名。这些区块可以被解析器识别为合法区块，但默认**不会被解析到任何数据字段**（相当于"注释区块"），适合用于备忘、草稿、设定集等。

**示例：**

```bash
export MEPHISTO_EXTRA_BLOCKS="设定集,草稿,废案"
```

此时以下区块可以被正确识别，不会报"未知区块"错误：

```meph
【设定集】
- 科技水平：文艺复兴
- 契约类型：灵魂交易

【草稿】
这段内容暂时保留，不影响叙事逻辑。
```

> **注意**：自定义区块的内容不会被引擎加载到任何数据结构中，因此不会影响 LLM 的上下文或规则匹配。如果你希望自定义区块真正参与叙事，需要修改源码（参见 `parse_block.go` 的 `parseBlocks` 函数）。
