# 梅菲斯特（Mephisto）

> **纯文本契约解析器 —— 将 `.meph` / `.mephisto` 文件解析为结构化数据，驱动规则引擎与大模型**
>
> ⚠️ 项目处于早期开发阶段（v0.2），文件格式尚未最终稳定，后续版本可能有调整。

---

## 这是什么？

梅菲斯特是一个**长线叙事引擎的底层核心**。它读取纯文本的 `.meph` 契约文件（用 `【】` 定义区块），解析为结构化数据，驱动规则引擎执行条件逻辑，为未来的大模型对话提供上下文。

**目前实现（v0.2）**：

- 读取 `.meph` / `.mephisto` 文件
- 按 `【区块名】` 分割内容
- 解析列表项（`- 键: 值`）和规则（`[名] if 条件 -> 动作`）
- **规则引擎**：条件判断、逻辑运算（`&&`、`||`）、骰子表达式（`roll(1d100)`）、互斥组（`[group:xxx]`）、变量替换（`{变量}`）
- **AST 预编译**：规则在加载时预编译，运行时直接求值，性能提升 10-100 倍
- **交互式对话界面**：多轮对话，实时触发规则
- **自动化测试**：契约测试（`sample.golden`）+ 引擎单元测试，保障代码健壮性
- 精准报错（带文件行号）

## 支持的语法

### 区块标题（必须独占一行）

```
【角色名】
【锚点】
【世界观】
【角色背景】
【开局场景】
【状态】
【规则】
【校验】
【记忆】
```

### 列表项（键值对）

```
- 键: 值
- 键：值          # 支持中文冒号
```

### 规则

```
[规则名] if 条件 -> 动作
```

支持：

- 原子条件：`情绪 == "暴怒"`、`堕落指数 > 60`
- 逻辑运算：`&&`、`||`、括号
- 骰子表达式：`roll(1d100) > 80`
- 互斥组：`[group:combat] 动作`

示例：

```meph
[攻击] if 情绪 == "暴怒" && 堕落指数 > 70 -> [group:combat] 全力攻击
[嘲讽] if 输入包含 "光之国" && 情绪 != "平静" -> 先嘲讽再回应
[狂笑] if roll(1d100) > 80 || 情绪 == "疯狂" -> 大笑
```

### 全局变量

`【状态】` 中的所有键（如 `{情绪}`、`{生命值}`、`{位置}`）和 `【角色名】` 的值，都是**全局变量**，可以在以下区块中使用：

- `【开局场景】` — 场景描述
- `【世界观】` — 世界设定
- `【角色背景】` — 角色历史
- `【规则】` 动作 — 规则触发时注入的内容

示例：

```
【角色名】
暗影行者

【状态】
- 生命值: 100
- 情绪: 冷静
- 位置: 暗影森林

【世界观】
{角色名}是暗影森林的守护者，拥有{生命值}点生命值。

【开局场景】
你站在{位置}中，{角色名}的记忆开始苏醒。

【规则】
[自我介绍] if 输入包含 "你是谁" -> 注入 "我是{角色名}"
```

输出时，`{角色名}` 会被替换为 `暗影行者`，`{生命值}` 替换为 `100`，`{位置}` 替换为 `暗影森林`。

### 注释

```
# 这是一条注释
```

## 快速开始

```bash
# 克隆项目
git clone https://github.com/yourname/mephisto.git
cd mephisto

# 运行
go run main.go data/sample.meph

# 或构建后运行
go build -o mephisto main.go
./mephisto data/sample.meph
```

## 8 个标准区块

| 区块           | 类型       | 必填 |
| :------------- | :--------- | :--- |
| `【角色名】`   | 单行文本   | ✅   |
| `【锚点】`     | 键值对列表 |      |
| `【世界观】`   | 多行文本   |      |
| `【角色背景】` | 多行文本   |      |
| `【开局场景】` | 多行文本   |      |
| `【状态】`     | 键值对列表 |      |
| `【规则】`     | 规则列表   |      |
| `【校验】`     | 键值对列表 |      |
| `【记忆】`     | 键值对列表 |      |
